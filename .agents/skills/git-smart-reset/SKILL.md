# Git Smart Reset Skill

Этот навык позволяет безопасно откатить последний коммит, сохранив при этом конкретные файлы, которые были в этом коммите.

## Когда использовать
- Вы сделали коммит в форк, но хотите "очистить" историю перед PR, оставив только часть файлов.
- Вы ошиблись с коммитом и хотите переделать его, сохранив наработки.

## Инструкции для ИИ (Antigravity)

Если пользователь просит откатить коммит и оставить файлы:

1. **Определить список файлов**, которые нужно сохранить.
2. **Выполнить жесткий откат** на один коммит назад:
   ```powershell
   git reset --hard HEAD~1
   ```
3. **Восстановить файлы** из состояния до отката (используя `ORIG_HEAD`):
   ```powershell
   git checkout ORIG_HEAD -- <пути_к_файлам>
   ```
4. **Подтвердить результат**: проверить, что файлы на месте и история откатилась.

## Скрипт для автоматизации

Для удобства этот процесс вынесен в скрипт `scripts/smart-reset.ps1`.

### Использование скрипта:
```powershell
.\smart-reset.ps1 "file1.ts", "file2.ts"
```

## Метрики успеха
- `git log -n 1` показывает предыдущий (оригинальный) коммит.
- Указанные файлы присутствуют в рабочей директории и имеют статус "Modified" или "Untracked".
