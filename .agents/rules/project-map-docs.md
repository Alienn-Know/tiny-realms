# Project Map Docs

Когда работа затрагивает файлы в `src/`, нужно поддерживать карту модулей в `docks/` в актуальном состоянии.

- Используй skill `project-map-docs` (@.agents/skills/project-map-docs/SKILL.md) для структуры и оформления.
- Зеркаль дерево `src/` внутри `docks/src/`.
- Убеждайся, что каждая сгенерированная папка содержит `README.md`.
- Для каждого исходного файла создавай отдельную карточку с тем же именем и добавленным `.md`, например `main.js.md`.
- Самодокументация по умолчанию должна создаваться на русском языке.
- Описания должны быть короткими, фактическими и без воды.
- Добавляй ссылки на дочерние модули и прямые исходные файлы.
- В карточках файлов описывай только: зачем файл нужен и по какому простому принципу он работает.
- В карточках файлов добавляй детальное описание именованных методов.
- В карточках файлов добавляй простую карту зависимостей методов через Mermaid.
- В карточках файлов добавляй ключевые константы, их значения и пояснение, для чего они нужны.
- Не пересказывай код построчно, если это уже очевидно из исходника.
- Добавляй Mermaid-стрелки для зависимостей и обратного использования.
- Если в документе есть разделы `Зависит от` и `Используется в`, добавляй туда простые Mermaid-схемы прямо внутри этих подпунктов.
- Для визуального контроля используй выразительные символы вроде `🌟`, `👤`, `👥`, `📄`, `🍎`, `🍑`, а не однотипные цветные кружки.
- Сохраняй проектный формат папки `docks/`.
