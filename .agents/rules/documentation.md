# Правила документирования кода (JSDoc)

При написании методов и объявлении переменных ВСЕГДА добавляйте JSDoc-комментарии, следуя этим правилам:

1. **Язык**: Обязательно на **русском языке**.
2. **Логический смысл**: Объясняйте не *как* код работает технически (алгоритм), а **что он делает логически** для игры (с точки зрения механик).
   - *Плохо*: "loop through array and add values"
   - *Хорошо*: "распределяет накопленные ресурсы по доступным складам, учитывая их лимиты"
3. **Конфиденциальность**: Не придумывайте предназначение, если оно не на 100% ясно из анализа Java-кода. Помечайте такие места как "неизвестно" или "требует уточнения".
4. **Переменные**: Каждая важная переменная должна иметь краткое описание её роли в игровом мире (например, "количество единиц провизии в ратуше").

```typescript
 * Организует процесс завершения строительства здания.

 * позволяя юнитам-добытчикам начать поставку сырья.
 */
public finalizeConstruction(): void {
    // ...
}
```
