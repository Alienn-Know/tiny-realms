---
description: Воркфлоу для безопасных экспериментов и рефакторинга (Shadow Branching)
---

Используй этот воркфлоу, когда нужно проверить сложную гипотезу, сделать рефакторинг ядра или внедрить новую крупную фичу, не ломая `main`.

### Шаги выполнения:

1. **Подготовка ветки**:
   - Создай временную ветку: `git checkout -b experiment/<name>`.
   - Запомни текущее состояние файлов.
2. **Реализация**:
   - Выполни необходимые изменения в коде.
   - Если проект поддерживает сборку, запусти `npm run build` или аналогичную команду проверки.
3. **Анализ и Отчет**:
   - Сравни изменения с оригиналом.
   - Подготовь отчет для пользователя:
     * Что было изменено?
     * Собирается ли проект (Build status)?
     * Есть ли визуальные или логические изменения (если проверял через браузер)?
4. **Решение**:
   - Спроси пользователя: "Переносим эти изменения в основную ветку или удаляем эксперимент?".
   - Если пользователь согласен -> `git checkout main` -> `git merge experiment/<name>`.
   - Если нет -> `git checkout main` -> `git branch -D experiment/<name>`.
