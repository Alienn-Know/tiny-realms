---
description: Мастер создания изолированных веток (Git Worktrees) для параллельной работы ИИ.
---
При запуске этого воркфлоу, ИИ берет на себя роль оркестратора и выполняет шаги строго по очереди, дожидаясь ответа пользователя после каждого вопроса:

1. **Шаг 1: Префикс.** Спроси у пользователя: "Введите базовое название (префикс) для папок веток (например, `townsmen`)." Дождись ответа.
2. **Шаг 2: Суффиксы или Описание.** Спроси: "Какие модули будем делать? Вы можете просто перечислить суффиксы (например, `ui audio economy`), ИЛИ описать задачу своими словами, и я сам разобью её на оптимальное количество веток и предложу их названия." Дождись ответа.
3. **Шаг 3: Подтверждение плана.** Если пользователь описал задачу абстрактно, составь список предлагаемых веток в формате `feature/<suffix>` и папок `../<prefix>-<suffix>`. Спроси, согласен ли он с таким разделением. Дождись ответа.
4. **Шаг 4: Выполнение.** Используя инструмент командной строки, создай worktree для каждой утвержденной ветки. Выполняй команды `git worktree add -b feature/<suffix> ../<prefix>-<suffix> main`.
5. **Шаг 5: Инструкции.** Разверни перед пользователем готовую текстовую инструкцию, которую он должен скопировать и отправить первому сообщением в каждый новый ИИ-чат, чтобы "запереть" агентов в их рамках (см. воркфлоу `strict-branch-agent`).
