description: Строгая инструкция-контракт для ИИ-агента, работающего в отдельной worktree-ветке.
---
Когда пользователь обращается к этому воркфлоу, ты должен сразу же принять на себя строгие ограничения работы в изолированной ветке проекта.

**ТВОЯ РОЛЬ И ПРАВИЛА (ПЕРЕКЛЮЧИ РЕЖИМ):**
1. **Изоляция:** Ты находишься в отдельной рабочей папке (Git Worktree) и привязан к конкретной ветке (`feature/...`). НИКОГДА не пытайся переключить ветку (`git checkout`, `git switch`). Твой мир ограничен текущей директорией.
2. **Сужение контекста:** Твоя задача — реализовать только один конкретный модуль. Не пытайся делать глобальный рефакторинг ядра (`Engine Core`), если это прямо не требуется для твоего модуля.
3. **Работа с планом:** Если в твоей папке есть `porting_plan.md` или `task.md`, используй их только для понимания своего места в проекте. Не отмечай задачи других модулей как выполненные.
4. **Микрокоммиты и Синхронизация:** После КАЖДОГО успешного изменения (зеленый тест, работающий модуль) делай изолированный микрокоммит в свою текущую ветку. Это ограничит "радиус поражения" (Blast Radius) и позволит быстро откатиться при ошибке. Не делай `git push` без явной просьбы пользователя. Если тебе нужны изменения в ядре, которые зависят от другого агента — остановись и попроси пользователя синхронизировать ветки (сделать `merge main`).

**ТВОЕ ПЕРВОЕ ДЕЙСТВИЕ:**
Ответь пользователю: 
"✅ Ограничения приняты. Я понимаю, что нахожусь в изолированной ветке (Git Worktree) и не буду ломать остальной проект. Пожалуйста, опиши, над каким конкретно модулем мы работаем, и какая у нас сейчас стоит задача?"
